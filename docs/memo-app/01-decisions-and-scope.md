# 決定事項とスコープ

## 1. 決定事項

現時点の決定事項は以下。

* MindHub_Appをベースにする
* 最初はPC用Webアプリとして作る
* 普段の思考メモも保存対象に含める
* データ保存にはSQLiteを使う
* 本文はMarkdown形式を基本にする
* Markdownプレビューを入れる
* ChatGPTとの連携はAPIではなくプロンプトコピー運用にする
* ChatGPT側に形式を記憶させない
* アプリ側にカテゴリ別の整理プロンプトを持たせる
* Git自動pushは初期実装では行わない
* Gitに上げたいメモはアプリ上で候補として選べるようにする
* Git候補メモはMarkdownとして書き出せるようにする
* 削除は完全削除ではなくアーカイブ方式にする
* プロジェクト名は自由入力と候補選択の両方に対応する
* タグも自由入力と候補選択の両方に対応する
* スマホ確認はスマホ閲覧用HTML/JSONで行う（当初の「GitHubアプリで代用する」方針は2026-07-06の追加決定で置き換え。`13-mobile-view-export.md` 参照）

## 1.4 追加決定事項（2026-07-06 配布・共有方針）

データ・配布物を三区分に分ける（詳細は `17-distribution-and-sharing.md`）。

採用。

* 自分用：完全版。Android APK版で自分が使用する。自分のメモ・テンプレート・家族用データ・家庭内の決まり事・個人的な知識データを含めてよい
* 家族用：家族は編集せず、Webページで閲覧する。家族間の決まり事・家庭内マニュアル・共有したい情報を表示する。自分用データから出力するが、家族に見せてよい情報だけを対象にする
* 配布用：知人などに渡す可能性があるため、自分用・家族用とは分離する。配布用リポジトリを別に作り、汎用の知識データ・汎用テンプレート・サンプルデータ・説明書だけを含める。個人情報・家族情報・家庭内の決まり事・私的メモは含めない。自分用の完全版をそのまま渡さない
* 同じリポジトリ内のディレクトリ分離ではなく、配布用は別リポジトリにする（更新の手間より情報混入リスク低減を優先）

将来候補。

* 配布用データだけを書き出すエクスポート機能

## 1.6 追加決定事項（2026-07-12 名称方針）

MindHub と FlowDock の呼び分けを以下のとおり整理する。今後の開発・AIへの依頼時はこの呼び分けに従う。

### 名称の定義

* **MindHub**：アプリ全体名。思考メモ、作業ログ、AI活用、プロンプト集、現場適応モードをまとめるハブ。アプリ全体の表向き名称は今後 MindHub に寄せる
* **FlowDock**：初期からある軽量メモ機能の説明名。スマホで素早くメモを流し込み、あとで整理につなげる入口。名称としては消さず、既存軽量メモ機能・旧スマホメモ機能の説明名として残す

### 機能系統と名称の対応

| 系統 | 名称・位置づけ |
|------|--------------|
| `/memo` 系（memosテーブル、`app/memo/*`） | FlowDock由来の既存軽量メモ機能 |
| `/notes` 系（notesテーブル、`app/notes/*`） | メモ管理機能 |
| `/workplace` 系（`app/workplace/*`） | 現場適応モード |

### 現時点で変更しないもの

名称方針はドキュメント上の呼び分け整理であり、以下は現時点では変更しない。

* app.json（`name: "FlowDock"`、`slug` / `scheme: flowdock`。android.package は com.ykjob.mindhub で確定済み）
* package.json
* ルート名（`/memo`、`/notes`、`/workplace` 等）
* DB（テーブル名・スキーマ）
* 実装コード（画面タイトル「FlowDock」等の表示文言を含む）

表示名・slug等を実際に MindHub へ変更するかどうか・その時期は未確定として扱う（変更する場合は影響範囲を確認のうえ別途判断）。

### 補足

* リポジトリ名は MindHub_App のまま
* `docs/flowdock_mvp_design.md` は FlowDock（既存軽量メモ機能）の仕様基準として従来どおり参照する
