# 画面構成と操作フロー

## 1. メイン画面（ホーム）

ホームは**MindHub全体の入口**とする（2026-07-13 Phase 15で役割を再定義。詳細設計の正本は `28-ui-ux-quality-improvement.md` §7）。

* MindHubタイトルと短い説明を置き、アプリ全体の目的が分かるようにする
* 主要機能への導線を優先順位つきで並べる：最重要＝クイックメモ（FlowDock由来の軽量メモ）・現場適応モード、主要＝メモ管理・プロンプト集（優先関係の正本は `28` §6.2）
* 設定は主要機能と同格に扱わず、補助位置（ヘッダー右側等）へ置く
* 最近のメモは最大3件程度に限定し、一覧全体は「すべて見る」から開く
* ホーム再構成は画面内容の変更であり、**既存ルート（`/`・`/memo`・`/notes`・`/workplace`・`/prompts`・`/settings`）は変更しない**

旧方針の記録：当初のメイン画面は「現在のメモ状況の把握」（最近更新メモ・Git候補メモ・プロジェクト別・カテゴリ別・未整理・一覧などの表示候補）を役割としていたが、この役割はPhase 15の入口再構成で置き換えた。メモ状況の把握はメモ管理一覧（`/notes`）側が担う。旧Phase 8ダッシュボード構想（集計・統計パネル）は将来候補のまま（`28` §12）。

## 6. ChatGPT整理プロンプトコピー操作

流れ。

1. カテゴリを選ぶ
2. 「ChatGPT整理プロンプトをコピー」を押す
3. クリップボードにコピーされる
4. コピー完了メッセージを表示する
5. ユーザーがChatGPTに貼り付ける
6. 整理結果を本文に貼り戻す

## 7. Markdown書き出し操作

流れ。

1. メモ詳細を開く
2. Git候補状態を確認する
3. Markdown書き出しを押す
4. 出力先パスを自動生成する
5. 必要なら保存先フォルダを指定する
6. Markdownファイルを書き出す
7. export_pathとexported_atを保存する

## 8. アーカイブ操作

流れ。

1. メモ詳細を開く
2. アーカイブを押す
3. 確認ダイアログを表示する
4. archived_atを保存する
5. 通常一覧から非表示にする

## 9. 現場適応モードへの入口（詳細は 21 を参照）

* メイン画面の導線に「現場適応モード」入口を置く（Phase 14実装済み）。
* 「既存の主要導線を壊さず、入口追加に留める」という制限はPhase 14実装時のものであり、**Phase 15のホーム再構成（§1・`28` §7）で置き換わった**。ホームは入口として再構成するが、現場適応モードへの導線と既存機能は維持する。
* 現場適応モード内の詳細画面フロー（場面選択・作業開始・詰まり記録・質問文作成・進捗報告作成・終業前メモ・翌朝再開導線・現場プロファイル）は `21-workplace-adaptation-flows-and-ui.md` を正本とする。
