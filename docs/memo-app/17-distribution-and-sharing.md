# 配布・共有方針（自分用・家族用・配布用の三区分）

追加日：2026-07-06

## 1. 目的

データと配布物を「自分用・家族用・配布用」の三区分に明確に分け、個人情報・家族情報の混入リスクを下げる。

端末別の運用方針（誰がどの端末で使うか）は `16-platform-and-distribution.md` を参照。本ファイルは「誰に何を渡すか・見せるか」の区分を定める。

## 2. 三区分の整理

| 区分 | 対象者 | 形態 | 含めてよいもの |
|------|--------|------|----------------|
| 自分用 | 自分 | Android APK版（完全版） | すべて（私的メモ・家族用データ・個人的知識含む） |
| 家族用 | 家族 | Web閲覧ページ（編集なし） | 家族に見せてよい情報だけ |
| 配布用 | 知人など | 別リポジトリ | 汎用データ・サンプル・説明書だけ |

## 3. 自分用（完全版）

* 完全版として運用する
* Android APK版で自分が使用する（`16-platform-and-distribution.md` 2章）
* 以下を含めてよい
  * 自分のメモ
  * テンプレート
  * 家族用データ
  * 家庭内の決まり事
  * 個人的な知識データ

## 4. 家族用（Web閲覧版）

* 家族は編集せず、Webページで閲覧する（`16-platform-and-distribution.md` 3章、`13-mobile-view-export.md`）
* 表示する内容
  * 家族間の決まり事
  * 家庭内マニュアル
  * 共有したい情報
* 自分用データから出力するが、家族に見せてよい情報だけを対象にする

### 4.1 公開範囲（2026-07-06決定）

* 家族間の決まり事・家庭内マニュアルは、公開GitHub Pagesには置かない
* 公開GitHub Pagesに出してよいのは、公開されても困らない内容だけに限定する
* 家庭内情報の共有手段は別途検討する。候補：非公開配置先、ローカルHTML共有、PDF、JSONインポート、将来のprivate向けPWA（どれから着手するかは `11-open-issues.md` 13章）

### 4.2 familyカテゴリと公開判定（2026-07-06決定）

* 家庭内情報用に `family` カテゴリを追加する（`04-categories-and-tags.md` 参照）
* ただし、公開可否はカテゴリだけで判定しない。visibilityを併用する
* category = family でも、visibility = private または visibility = family のものは公開GitHub Pagesには出さない

## 5. 配布用（別リポジトリ）

### 5.1 方針

* 知人などに渡す可能性があるため、自分用・家族用とは分離する
* 配布用リポジトリを別に作る
* 自分用の完全版をそのまま渡さない

### 5.2 含めるもの

* 汎用の知識データ
* 汎用テンプレート
* サンプルデータ
* 説明書

### 5.3 含めないもの

* 個人情報
* 家族情報
* 家庭内の決まり事
* 私的メモ

### 5.4 別リポジトリにする理由

* 同じリポジトリ内のディレクトリ分離より、配布用は別リポジトリにする方が安全
* 更新は手間になるが、情報混入リスクを下げることを優先する

### 5.5 配布物の範囲（2026-07-06決定）

初期に優先する配布物。

* Web / PWA
* JSON
* 説明書

将来候補（初期必須にはしない）。

* 配布用APK
* ソース公開

## 6.1 現場適応モード情報の非混入（2026-07-09 追加）

現場適応モード（`20`〜`23`）で扱う現場情報・場面メモは、三区分いずれの共有対象にも含めない。

* 配布用（別リポジトリ）：汎用データのみ。現場情報は含めない
* 家族用：家庭内情報のみ。現場情報は含めない
* 公開GitHub Pages（`13-mobile-view-export.md`）：現場情報を出力対象にしない

守秘方針の正本は `23-workplace-adaptation-security-and-portfolio.md`。

## 6.2 OS共有（Phase 16C）と三区分の区別（2026-07-23 追加）

Phase 16C（`33-external-handoff-and-sharing.md`）の**ユーザー操作によるOS共有**は、本ファイルの三区分（公開／家族用／配布用）とは別概念である。

* OS共有は公開用データ生成（公開GitHub Pages・家族用Web閲覧・配布用リポジトリ）ではない
* ユーザーが確認した1件の文章を、その場で外部アプリへ渡す操作にすぎない
* 現場適応の質問・報告を外部共有する場合は、`23`／`33` の守秘確認（守秘チェック必須）を優先する

三区分の既存ルール（公開範囲・家族用・配布用に含めてよい/いけない情報）は本追記で緩めない。
