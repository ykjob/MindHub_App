# スマホ用プロンプト集HTMLとmobile-inbox運用

追加日：2026-07-06

## 1. スマホ用プロンプト集HTML

### 1.1 目的

外出先のスマホから、よく使う整理プロンプトをすぐコピーできるようにする。

DB同期なしで、スマホからすぐ使えるプロンプト集を用意する。

外出先ではMindHub_Appへ直接保存せず、ChatGPT / Gemini連携でタスク化する。

### 1.2 初期実装方針

最終的には、直近数か月で使っているプロンプトを広く載せたいが、初期実装では10個に絞る。

2026-07-06更新：実装時に方針を拡張し、既存のメモ整理プロンプト（カテゴリ別）に加えて、追加プロンプト31本（タスク・予定整理4、開発・AI作業8、思考整理・行動化5、就活5、生活・家庭共有5、投資・検証4）を収録した。§1.3の初期10プロンプトはすべてこの中に含まれる。今回見送り：Claude Code報告確認、リポジトリ状況整理、検証結果レビューの3本。2026-07-07にfamilyカテゴリが追加され、カテゴリ別は11本・計42本になった（§1.7参照。当初「計41本」と記載していたが、familyカテゴリ追加後の正しい合計は計42本）。

### 1.3 初期10プロンプト候補

#### 1. 雑メモを時間帯別タスクリストにする

用途。

* 手書きメモ、音声入力、思いつきメモを整理する
* 朝・昼・夕方・夜などの時間帯別に分ける
* 今日やること、後でやること、やらなくていいことを分ける

#### 2. Googleタスク投入用に整形する

用途。

* ChatGPTで整理した内容をGeminiへ渡しやすい形にする
* タスク名、期限、メモ、優先度をリスト化して分ける
* Googleタスクへ登録する前の整形を行う

#### 3. Googleカレンダー投入用に整形する

用途。

* 予定、移動、通院、買い物、家事などをカレンダー登録用に整理する
* 開始時刻、終了時刻、場所、メモを分ける
* 時刻が曖昧なものは確認必要として残す

#### 4. 今日の優先順位とやらないことを整理する

用途。

* やることが多い日の優先順位を決める
* 今日やるべき、後日でいい、今はやらないに分ける
* 就活、開発、家事、PEAD、ココロコールなどが混ざった時に使う

#### 5. 思考メモを整理する

用途。

* ダーッと書いた不安・焦り・考えを整理する
* 課題、決めること、今やること、後で考えることに分ける
* MindHub_Appに保存する思考メモの元にする

#### 6. 作業ログ・次回再開メモにする

用途。

* ChatGPTやClaude Codeとの作業内容を保存用にまとめる
* 今回やったこと、決まったこと、未完了、次にやることを整理する
* 「今日はここまで」の一旦区切り時の保存ログとして使う

#### 7. Claude Code実装依頼プロンプトにする

用途。

* 仕様、目的、やらないこと、変更範囲、完了条件を整理する
* Claude Codeへ投げる前の依頼文を作る
* 実装後確認、作業ログ更新、commit/push禁止などの条件を含める

#### 8. Codexレビュー依頼プロンプトにする

用途。

* 実装済み差分をレビューしてもらう
* 既存機能への影響、DB、マイグレーション、セキュリティ、型、エラーハンドリングを確認させる
* レビュー専用、勝手に修正しない、commit/pushしない条件を含める

#### 9. エラー対応メモにする

用途。

* エラー全文、発生状況、試したこと、原因候補、解決方法を整理する
* Git、PowerShell、Expo、SQLite、GitHub Actions、PEADなどのトラブル用
* 後から同じエラーを見返せるようにする

#### 10. 就活メモ・面接準備に整理する

用途。

* 職務経歴書、自己PR、転職理由、面接で話す内容を整理する
* 介護経験、IT訓練校、アプリ開発、業務改善経験を話しやすい形にする
* 個人情報が入りやすいため、Git候補false前提で扱う

### 1.5 プロンプトの再利用

スマホ用プロンプト集HTMLのプロンプトは、アプリ内のChatGPT整理プロンプト（DB上のテンプレート）と同様の内容を再利用できるようにする（`06-chatgpt-prompt-copy.md`、`12-template-db-management.md` 参照）。

### 1.6 生成方式（2026-07-06決定）

短期対応と将来対応の二段階にする。

* 短期：note_templatesのDB化（Phase 8）を待たず、現在コード内に固定されているプロンプト定義（`src/features/notes/chatgptPrompts.ts`）から、docs配下へスマホ閲覧用HTMLを生成する
* 将来：note_templates実装後に、出力元をDBへ切り替える

これにより、プロンプト集HTMLはテンプレートDB管理（Phase 8〜9）の完成を待たずに先行してスマホで使えるようにする。

### 1.7 追加プロンプト定義（2026-07-06実装）

* 追加プロンプトは `src/features/notes/mobilePrompts.ts` にコード固定で定義する（データ専用モジュール。アプリ画面からはimportせず、メモ作成画面のカテゴリ選択肢には混ぜない）
* prompts.htmlは7セクション構成：メモ整理プロンプト（カテゴリ別）／タスク・予定整理／開発・AI作業／思考整理・行動化／就活／生活・家庭共有／投資・検証
* 2026-07-07更新：familyカテゴリ追加によりカテゴリ別は11本・計42本になった。セクション見出しの件数表示・絞り込み中の表示件数を追加。familyカテゴリのカードは「private / family用」バッジ表示
* 生活・家庭共有系の5本には「private / family用」バッジと「公開GitHub Pagesには載せない前提」の注記を付ける（プロンプト本文自体に個人情報・家庭内情報は含めない）
* 将来 note_templates 実装後は、category_type='template' の複数テンプレートとしてseedし、出力元をDBへ切り替える

### 1.8 Phase 15 UI・UX改善との関係（2026-07-13追記）

* Phase 15では、既存プロンプト本文（42件）・分類・生成ロジック（`promptHub.ts` / `generate_prompt_hub.mjs`）を変更せず、アプリ内プロンプト一覧画面（`app/prompts`）に**利用状況から選ぶ入口を追加する可能性がある**
* 画面別改善の正本は `28-ui-ux-quality-improvement.md` §10、共通UI基盤は `29-ui-design-system.md`、検証は `30-ui-validation-checklist.md`
* 状況別分類は未確定（`11-open-issues.md` §16）。この追記は本ファイルが正本とするプロンプト本文・生成ロジックを変更するものではない

#### 1.8.1 状況別入口の確定（2026-07-14実装＝Phase 15バッチ5）

* 状況別入口6件を確定（PROMPT-01）：**問題を整理する／エラー・不具合を整理する／予定・やることを整理する／作業結果をまとめる／引き継ぎを作る／開発作業を依頼・確認する**
  * 入口名・ID対応は入口名からの広い推測ではなく、**既存プロンプト本文が実際に行う処理**に合わせて確定した（同日、当初実装から修正）。旧「エラーを相談する」はAIへ相談ではなくエラー・不具合・検証ログを所定形式へ整理する処理のため「エラー・不具合を整理する」へ。旧「仕様・実装を確認する」の `design`/`research` は仕様確認ではなく設計メモ・調査メモへの整形のため入口から除外
* 状況別入口は**既存42件から目的に近いものを絞り込む補助フィルター**であり、42件を削減・置換・分類し直すものではない。既存7セクション（42件すべてを体系別に確認する通常導線）・検索（名称/ID/セクション名から探す通常導線）は不変で、**入口に登録されていないプロンプトも「すべて」・既存セクション・キーワード検索から従来どおり利用できる**
* **全42件を6入口へ無理に割り当てない**。就活・投資、および生活・家庭共有のうち `family_manual`・`household_rule` は入口に登録せず、既存セクション・検索から利用する。一方で生活・家庭共有カテゴリを一律に除外はせず、入口の目的に自然に一致する `wife_schedule`・`outing_plan`・`shopping_memo` は「予定・やることを整理する」に個別対応させる
* 実装は `app/prompts/index.tsx` 内の固定IDマッピング（画面側のローカル定義）のみ。**本ファイルが正本とするプロンプト本文・ID・件数（計42本）・所属セクション・生成処理（`promptHub.ts` / `mobilePrompts.ts` / `chatgptPrompts.ts` / `generate_prompt_hub.mjs`）・HTML版（`docs/mobile-view/prompts.html`）は一切変更していない**（`mobilePrompts.ts` への状況タグ追加や `PromptEntry` の型拡張は行わない）
* 同一プロンプトの複数入口重複は、どちらの目的から探しても自然なものだけ許可する（`next_one_action`／`validation_log`／`worklog_close_summary`／`worklog`）。件数をそろえるための弱い関連は追加しない
* 今回、新規プロンプトは追加していない。現行42件で不足する可能性がある後続候補は §15（将来候補）と `11-open-issues.md` §16に記録し、バッチ5では追加せず別タスクで判断する
* 詳細な入口→ID対応は `28-ui-ux-quality-improvement.md` §10.1、確定経緯は `11-open-issues.md` §16

## 2. mobile-inbox運用

### 2.1 方針

mobile-inboxは、外出先でスマホから入力したメモを後で整理するための考え方として採用する。

ただし、初期実装では専用保存場所は作らない。

### 2.2 初期運用

```text
スマホで雑メモを書く
↓
そのままChatGPTへ送信する
↓
構造化テンプレートで整理してもらう
↓
必要に応じてGeminiへ渡してGoogleタスクやGoogleカレンダーに入れる
```

### 2.3 初期実装でやらないこと

* `mobile-inbox/*.md` のようなファイル保存
* スマホからMindHub_AppのSQLiteへ直接保存
* Codexによる日常的なスマホ入力取り込み
* GitHub経由の自動取り込み

### 2.4 将来候補

必要になった場合のみ、以下を検討する。

* `mobile-inbox/` フォルダへの保存
* GitHub経由の一時メモ保存
* mobile-inboxをMindHub_Appに取り込む機能
* スマホ入力欄付きの専用HTML
