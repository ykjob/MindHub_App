# 現場適応モード 画面フローと5場面導線

本ファイルは、現場適応モードの5場面導線と画面フローの正本とする。
方針・スコープは `20-workplace-adaptation-overview.md`、DB・保存挙動は `22-workplace-adaptation-data-and-integration.md`、守秘表示は `23-workplace-adaptation-security-and-portfolio.md` を参照する。
画面の横断正本は `08-ui-flow.md`。本ファイルは現場適応モード固有の導線を定義し、`08` には入口導線のみ追記する。

## 1. 全体導線

### 1.1 アプリ起動からの導線

1. アプリ起動 → ホーム（既存メモ一覧）
2. ホームのヘッダー、または既存の「ノート」「プロンプト集」導線と並ぶ位置に「現場適応モード」入口を追加する
3. 現場適応モード → 場面選択（作業開始／詰まり／質問／報告／終業前）と現場プロファイル確認

既存導線（`app/memo/*`、`app/notes/*`、`app/prompts`）は変更しない。現場適応モードは入口を1つ追加するのみとし、既存の主要導線・既存機能を壊さない（新規ルート追加は可）。

※2026-07-13追記：「入口を1つ追加するのみ」はPhase 14実装時の制限であり、**Phase 15のホーム再構成（`28` §7、`08` §1）で置き換わった**。ホームはMindHub全体の入口として再構成されるが、現場適応モードへの導線は維持され、現場適応モードの5場面・保存条件・コピー条件・守秘条件（`23`）は変更しない。

### 1.2 ホームから現場適応モードへの入口

* 既存ホームへの変更は入口ボタン追加のみに留める（※Phase 14実装時の制限。2026-07-13以降はPhase 15のホーム再構成で置き換え。上記1.1の追記参照）
* 入口から入った直後に「今日の場面」を選べるようにする（毎日の習慣化を優先）
* 現場適応入口画面（`/workplace`）自体の情報階層改善（一日の流れ3区分・動作種別表示）は `28` §9 を正本とする

## 2. 5場面モデル（正本）

現場適応モードの5場面は以下とする。

1. 作業開始
2. 詰まり記録
3. 質問文作成
4. 進捗報告作成
5. 終業前メモ

「翌朝の再開メモ」は5場面には**数えない**。終業前メモの出力を翌朝に表示し、次の作業開始へつなげる補助導線として扱う（`4. 翌朝の再開導線` 参照）。

MVPの優先順位（`20` 7.1）。

* 優先実装：作業開始／詰まり記録／終業前メモ（＋翌朝再開導線）
* 次点：質問文作成／進捗報告作成

## 3. 場面別の設計

各場面は「入力画面 → 出力画面」を基本とする。入力は短く、選択式・テンプレート式を優先する。出力はコピーを基本とし、保存は任意（保存時の既定は `23`、保存可否の最終判断は `11-open-issues.md`）。

### 3.1 作業開始（優先実装）

* 入力項目：今日の作業／完了条件／触る範囲（触ってよいファイル・機能）／確認事項（先に聞くこと）
* 出力：着手メモ（今日の作業サマリ）
* 補助：現場プロファイルの命名規則・ブランチ運用を参照表示

### 3.2 詰まり記録（優先実装）

* 入力項目：状況（何をしていたか）／試したこと／確認したいこと／エラー内容（任意）
* 出力：整理済み詰まりメモ
* 意図：未経験者の抱え込み防止。詰まりを言語化し、質問文作成（3.3）へ接続できる

### 3.3 質問文作成（次点）

* 入力：詰まり記録（3.2）の内容＋前提（現場・使用ツール）を引き継ぐ
* 出力：質問文テキスト（相手が答えやすい形。目的・試したこと・聞きたいこと・期待する回答）
* 連携：整理プロンプト（`06`）・プロンプト集（`14`）の質問文テンプレートを利用できる

### 3.4 進捗報告作成（次点）

* 入力：完了したこと／未完了のこと／次にやること／ブロッカー（任意）
* 出力：進捗報告文テキスト（現場プロファイルの報告形式に合わせる）
* 連携：報告形式（箇条書き／所定フォーマット）は現場プロファイルから取得

### 3.5 終業前メモ（優先実装）

* 入力項目：今日やったこと／未完了事項／明日最初にやること／再開のための補足
* 出力：再開メモ（翌朝そのまま読めば再開できる粒度）
* 接続：翌朝の再開導線（4章）で表示する

## 4. 翌朝の再開導線（5場面とは別）

翌朝の再開メモは独立した場面ではなく、終業前メモ（3.5）の出力を翌朝に見せる補助導線である。

* 現場適応モードを開いた際、直近の終業前メモ（再開メモ）を上部に表示する
* 「この続きから作業開始」で作業開始（3.1）へ遷移し、当日の作業整理につなげる
* これにより「終業前メモ → 翌朝表示 → 作業開始」の1日ループを閉じる

## 5. 画面種別の区別

| 画面種別 | 役割 | 主な要素 |
| --- | --- | --- |
| 入力画面 | 場面ごとの短い入力 | 選択式・テンプレート式・自由入力（最小限） |
| 表示画面 | 保存済み情報の参照 | 現場プロファイル、過去の再開メモ・詰まりメモ |
| 出力画面 | 生成テキストの受け取り | コピー、（任意で）保存 |

* 入力画面は既存 `notes` 作成画面のスタイル（`#2563EB` 基調・カード型・StyleSheet）を踏襲する（※2026-07-13更新：Phase 15以降は「既存スタイルの無条件踏襲」ではなく、`29-ui-design-system.md` の共通UI基準（トークン・色の役割・移行ルール）を優先する。検証・デモ再演は `30` を正本とする）
* 出力画面のコピーは既存のクリップボード実装（`app/prompts` と同じ `copyToClipboard`）を再利用する

## 6. 現場プロファイル画面

現場ごとのルールを保存・表示する画面。データ構造・保存方式は `22` を正本とする。

* 表示項目：現場名／使用ツール／質問先／報告先・報告形式／ブランチ運用／コミットルール／命名規則／よく使う資料の場所／保存禁止情報に関する注意
* 初期実装は単一現場でも成立させる。複数現場切り替えは将来前提（`22`）
* 具体的な顧客名・非公開URLは保存しない前提を画面上でも明示する（`23`）

## 7. テンプレート・プロンプト連携

* 質問文・報告文の生成には、整理プロンプト（`06`）・プロンプト集（`14`）のテンプレートを利用する
* 現場適応モード専用の追加プロンプトが必要な場合、本文の正本は `06` / `14`。本ファイルでは「どの場面でどのテンプレートを使うか」の対応のみ定義する
* 生成テキストは架空データでの利用を推奨（ポートフォリオ用途。`23`）

### 7.1 開発リファレンスとの関係（別機能・補助参照）

* 開発リファレンス（`24-development-reference.md`）は現場適応モードとは**別機能**であり、5場面には数えない
* 詰まり記録（3.2）・質問文作成（3.3）などで手が止まったとき、横から手順・コマンド・最小コード例を参照する補助導線として扱う
* 初期実装では現場適応モードの画面から開発リファレンス専用UIへは接続しない。既存 `notes` の `command` カテゴリ／タグ運用で参照する（`24` §6）。専用画面・横断参照導線は将来候補（`24` §7）
