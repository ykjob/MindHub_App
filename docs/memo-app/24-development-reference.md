# 開発リファレンス 方針

本ファイルは「開発リファレンス」機能の方針・目的・スコープの正本とする。
開発リファレンスは現場適応モード（`20-workplace-adaptation-overview.md`〜`23`）とは**別機能**であり、現場適応モードの中核機能ではない。既存notesの `command` カテゴリ／タグ運用の上に成り立つ補助参照の位置づけとして扱う。

## 1. 目的

開発リファレンスは、単語を覚えるための辞書ではない。
作業中に迷ったときに、手順・コマンド・最小コード例をすぐ確認し、集中力を切らさずに作業へ戻るための参照集である。

* 目的は「暗記」ではなく「即座に見返して手を止めない」こと
* 作業中に手が止まった瞬間、横から短時間で確認して作業へ復帰できることを最優先にする
* 網羅性・体系性・学習教材化は目的にしない（それは公式ドキュメント・書籍の役割）

## 2. 現場適応モードとの関係

* 現場適応モードは、作業開始・詰まり記録・質問文作成・進捗報告作成・終業前メモの**5場面**を支援する（`20` §5・§7.1、`21` §2）
* 開発リファレンスは、それらの場面で手が止まったときに**横から参照する補助導線**として扱う
* 開発リファレンスは現場適応モードの5場面には数えない。5場面モデル（`21` §2）や翌朝再開導線（`21` §4）を変更・拡張するものではない
* 位置づけの優先順位：現場適応モードの5場面支援が主、開発リファレンスは詰まったときの参照補助
* 守秘の考え方は現場適応モードと同じ方向（現場固有情報・機密は登録しない。§4・§5、`23` §3.1）だが、開発リファレンスは現場適応モード専用の守秘既定強制（`visibility=private` / `is_git_candidate=false` の上書き。`23` §6.1）とは独立して、既存notesの `command` カテゴリ運用に従う

## 3. 対象にするもの

作業中に見返して、コピペまたは短い確認で作業へ戻れるものを対象にする。

* よく使う確認手順
* よく使うコマンド
* 関数・メソッドの最小例
* ライブラリの最小利用例
* エラー時の確認順
* 質問前チェック（詰まったとき何を確認してから聞くか）
* 報告前チェック（報告前に何を揃えるか）
* 会話でよく出る一般IT用語の短い説明

## 4. 対象にしないもの

以下は開発リファレンスの対象にしない。

* AIが重要と言っただけの用語（実際にまだ使っていないもの）
* まだ実際に使っていない知識・先回りの知識
* 主要ライブラリの網羅的な関数一覧
* 公式ドキュメントの丸写し
* 長文解説・体系的な教材
* 現場固有のシステム名・顧客名・内部URL・業務手順（`23` §3.1 の職場情報の範囲に該当）
* 機密情報を含むコマンドや設定値（認証情報・接続文字列・内部エンドポイント等）

## 5. 登録基準

以下のいずれかに当てはまるものだけを登録する。基準を満たさないものは登録しない。

* 実際に作業中に手が止まったもの
* 2回以上調べたもの
* 1分以内に見返したいもの
* コピペまたは短い確認で作業へ戻れるもの

登録しない例。

* AIが「重要」と言っただけで、自分ではまだ使っていない用語
* 「いつか使うかもしれない」知識
* 網羅目的で集めた関数一覧・用語集

## 6. 初期実装方針

初期実装では専用DB・専用管理画面は作らない。既存機能の運用で扱う。

* 既存 `notes` の `command` カテゴリ（`04-categories-and-tags.md` §1「コマンド・手順」type=command）で扱う
* または既存notesのタグ運用で扱う（例：`ref` などの参照用タグ。具体的なタグ命名は未確定。`11-open-issues.md` で管理）
* 新テーブル追加・schema変更は行わない（`03-data-model.md` の schema_version 方式に従う。当面は追加しない）
* 専用画面は作らず、既存 `/notes` の一覧・検索・絞り込み・Markdownプレビュー・コピーを利用する
* 用語・関数・ライブラリの大量seedは作らない。実際に手が止まったものを1件ずつ登録する運用から始める
* Git候補・公開の扱いは既存 `command` カテゴリの初期値（`04` §2、Git候補true）に従う。ただし現場固有情報・機密は§4のとおり登録しないため、公開しても問題ない汎用的な内容に限る

## 7. 将来候補

初期実装では行わない。必要性が実運用で確認できた場合に検討する。

* 開発リファレンス専用の一覧・検索画面（`command` 系メモに特化した参照UI）
* 専用テーブル・専用データ構造（`03-data-model.md` を正本に schema_version 方式で検討）
* 現場適応モードの詰まり記録・質問文作成の画面からリファレンスへ横断参照する導線
* よく使うコマンド・最小コード例のスマホ閲覧（公開配布とは分離。`13-mobile-view-export.md` / `17-distribution-and-sharing.md` の守秘・公開範囲に従う）
* 一般IT用語の短い説明のテンプレート化（`12-template-db-management.md` の note_templates 実装後）

いずれも実際に「手が止まる回数が多い」「既存notes運用では探しにくい」と確認できてから着手する。先回りで作らない。
