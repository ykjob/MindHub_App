# FlowDock ロードマップ

最終更新：2026-06-26

---

## MVP（現在進行中）

### 目標

スマホでメモを書き、SQLiteに保存し、GitHubへ手動アップロードできる状態にする。

### 対象機能

- メモ作成（本文・カテゴリ）
- SQLite保存
- メモ一覧表示
- メモ詳細表示
- メモ編集
- 論理削除（削除確認ダイアログあり）
- カテゴリ選択（6種類）
- GitHubアップロード（手動・1件ずつ）
- GitHub設定保存（owner / repo / branch）
- GitHub token保存（SecureStore）
- GitHubアップロード状態表示（4状態）
- GitHubアップロード失敗時のエラー保存・表示
- 再アップロード機能
- AIアウトプット用テーブル作成（使用はしない）
- Google出力状態カラム確保（使用はしない）

### 対象外（MVP後に回す）

- Markdownプレビュー
- 自動同期
- 複数端末同期
- 通知
- AI整理処理
- Googleタスク出力
- Googleカレンダー出力
- 設計書生成
- プロンプト生成

---

## MVP後フェーズ1：AI整理

### 目標

Claude APIと接続し、メモを整理・要約・タスク候補生成できるようにする。

### 対象機能

- Claude API接続
- メモ本文をAIに渡して整理結果を取得
- AI整理結果の保存（ai_outputsテーブル活用）
- 要約生成・表示
- タスク候補生成・表示
- スケジュール候補生成・表示
- AIアウトプット種別管理（summary / task_schedule / prompt / spec / review）

---

## MVP後フェーズ2：Googleタスク出力

### 目標

AI整理で生成したタスク候補をGoogleタスクへ出力できるようにする。

### 対象機能

- Google認証（OAuth）
- Googleタスク作成
- Google出力状態管理（not_exported / exported / failed）
- 出力結果の確認UI

---

## MVP後フェーズ3：Googleカレンダー出力

### 目標

AI整理で生成したスケジュール候補をGoogleカレンダーへ出力できるようにする。

### 対象機能

- Googleカレンダーイベント作成
- カレンダー出力状態管理
- 出力結果の確認UI

## 技術的負債・後回し対応候補

| 項目 | 理由 |
|------|------|
| 自動同期 | MVPでは手動アップロードに集中 |
| Markdownプレビュー | 表示の複雑さを避けるため後回し |
| エラーリトライ自動化 | まずは手動再アップロードで対応 |
| テスト自動化 | MVP後に整備 |
